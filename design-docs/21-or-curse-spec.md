# 21 or Curse 仕様書（MVP）

## 1. ゲーム概要

- **ジャンル**: ワンカード判断型スコアアタック
- **コンセプト**: 「21に近づけたい。でも、捨て札の呪い（効果）が次の判断に影響する」
- **プレイ感**: ブラックジャックの「引く / 止める」に、
  「捨てて次を操作する」判断を追加した短時間ゲーム
- **想定プレイ時間**: 1プレイ 1〜3分

---

## 2. 勝敗・目的

- 目的は **バースト（合計値 > 21）せずに高スコアを獲得すること**。
- バーストした時点でゲーム終了、スコアは0。
- プレイヤーが任意終了（スタンド）した場合、
  その時点の合計値と倍率から最終スコアを計算する。

---

## 3. 基本ルール

### 3.1 カード

- 標準52枚デッキを使用（ジョーカーなし）。
- カードの数値:
  - 2〜10 = 額面通り
  - J/Q/K = 10
  - A = 11（MVPでは固定値）

### 3.2 ターン進行

1. 山札からカードを1枚公開する。
2. プレイヤーは次の3択を行う。
   - **取る（Take）**: 公開カードを合計値に加算
   - **捨てる（Discard）**: 公開カードを捨て、該当する捨て効果を発動
   - **勝負終了（Stand）**: その場でゲーム終了しスコア確定
3. 合計値が21を超えたら即バースト（敗北）。
4. バーストせず継続する場合、次ターンへ。

### 3.3 終了条件

以下のいずれかでゲーム終了:

- プレイヤーがStandを選択
- 合計値が21を超える（バースト）
- 公開ターン数が上限に到達（MVPでは10ターン）

---

## 4. 捨て札効果（Curse / Bless）

> MVP方針: 効果は**次の1枚のみ**に適用され、適用後に消える。

公開カードを捨てたとき、カード種別に応じて効果が付与される。

- **2〜5を捨てる**: 次に公開されるカードの見かけ値 `+1`
- **6〜9を捨てる**: スコア倍率 `+0.2`（上限 x2.0）
- **10/J/Q/Kを捨てる**: 次ターンの公開カードを先読み表示
- **Aを捨てる**: 現在の合計値 `-3`（下限0）

### 4.1 優先順位・競合解決

- 同時に保持できる「次の1枚効果」は1種類のみ。
- 新しい次枚効果が付与されたら、古い次枚効果は上書き。
- 倍率ボーナスは即時反映、効果時間なし（恒久）。
- 先読み効果は「次ターン開始時に1回だけ表示」し、そのターン終了で消滅。

---

## 5. スコア計算

- 初期倍率: `x1.0`
- 倍率上限: `x2.0`
- バースト時スコア: `0`
- 非バースト終了時:

```text
最終スコア = floor(合計値 × 倍率)
```

### 5.1 例

- 合計値19、倍率x1.4でStand
  - `floor(19 × 1.4) = 26`
- 合計値22（バースト）
  - `0`

## 6. UI/UX仕様（スマホ前提）

## 6.1 入力操作

- **右スワイプ**: 取る（Take）
- **左スワイプ**: 捨てる（Discard）
- **下スワイプ**: 勝負終了（Stand）
- 代替として、同等の3ボタンUIも用意（PC操作向け）

### 6.2 常時表示HUD

- 現在合計値
- 現在倍率
- 残りターン
- 次ターン効果（+1 / 先読みなど）
- 直前の行動ログ（例:「8を捨てた → 倍率 +0.2」）

### 6.3 画面遷移

1. **タイトル画面**
   - Start
   - ルール確認
2. **プレイ画面**
   - カード提示
   - スワイプ/ボタン入力
3. **リザルト画面**
   - 合計値
   - 最終倍率
   - 最終スコア
   - バースト有無
   - リトライ

## 7. 状態管理仕様（実装用）

ゲーム状態（例）:

```ts
interface GameState {
  deck: Card[];
  turn: number;                // 1..10
  maxTurns: number;            // 10
  total: number;               // 現在合計値
  multiplier: number;          // 1.0..2.0
  nextCardModifier: number;    // 0 or +1
  peekNextCard: boolean;       // 先読みフラグ
  isBurst: boolean;
  isFinished: boolean;
  finalScore: number | null;
}
```

ターン処理の要点:

1. 必要なら先読み表示
2. 現在カード公開
3. プレイヤー入力反映（Take/Discard/Stand）
4. バースト判定
5. 終了条件判定
6. 次ターンへ

---

## 8. バランス方針（MVP）

- 序盤の「捨て得」一辺倒を防ぐため倍率上限を設定。
- 複雑化を避けるため、継続効果は最小化（次の1枚のみ）。
- A効果を単純化し、直感的に扱えるようにする。

将来調整候補:

- 連続捨てペナルティ（例: 3連続Discardで倍率-0.1）
- Aの値を可変（1/11切り替え）
- デッキ構成の変更（ミニデッキ化）

## 9. 受け入れ基準（Acceptance Criteria）

- [ ] 21超過で即終了しスコア0になる。
- [ ] Standで任意終了し、`floor(total * multiplier)`でスコア確定する。
- [ ] 6〜9のDiscardで倍率が+0.2され、x2.0を超えない。
- [ ] 2〜5のDiscard効果は次の1枚にのみ適用される。
- [ ] 10/J/Q/KのDiscardで次カード先読みが1回だけ動作する。
- [ ] AのDiscardでtotalが3減り、0未満にならない。
- [ ] 最大10ターン到達で自動終了する。
